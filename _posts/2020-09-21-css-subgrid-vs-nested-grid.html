The CSS subgrid is a new feature coming to browsers. It's already supported by Firefox, but the Chromium team is also working on the implementation. The CSS subgrid allows grid items to adopt the grid tracks and grid lines of the grid container, but only for the area they're spanning over.

By default, the children of grid items are not part of the grid layout, so without a subgrid feature, you need to create a nested grid and recalculate the grid tracks if you want to use the same grid layout for one or more grid items — however a nested grid and a subgrid are still not the same thing.

What is CSS subgrid?

CSS subgrid is a value you can add to the grid-template-columns and grid-template-rows properties, for instance:

.grid-item {
	display: grid;
	grid-template-columns: subgrid;
	grid-template-rows: subgrid;
}

Using this syntax, .grid-item will adopt the grid tracks (rows and columns) of the grid container.

What is a nested grid?

A nested grid replicates the grid tracks of the parent grid, however it's still an independent grid.

You can use the inherit value for the grid-template-columns and grid-template-rows to create a nested grid, as follows:

.grid-item {
	display: inherit;
	grid-template-columns: inherit;
	grid-template-rows: inherit;
}

The difference between a subgrid and a nested grid

While a nested grid is still independent from the parent grid (we have two grids), a subgrid is the part of it (we have one grid) as the parent grid shares its rows, columns, and lines with it.
